What you see here

What you see here is an example of a web page that is generated automatically using a static website generator called HUGO. Check out GoHugo for yourself.

As a static website generator, HUGO has many competitors. Jamstack (Jamstack 2021) lists market data for several competitors, including Next.js, Gatsby, and Jekyll. I consider static website generators a commodity.

At the same, hosting providers who target static websites are also a commodity. I wouldn’t be surprised if major Content Delivery Networks (CDNs) leap-frog the hosting providers altogether. Acharya for Geekflare (Acharya 2021) lists his Top-10 providers to include Netlify, AWS (of course), GitHub Pages, Google Cloud, and Render. I am using Render for my static website because their cost is free.

The idea behind a static website generator is that constructing a website is declarative not procedural. Because the process is declarative that means everything is driven from text files that one manages through GitHub, which is why GitHub is offering GitHub Pages.

My process

I write my web page using R-Studio, which is a free text-based editor that targets the data science community. I like R-Studio (actually, the R-Studio ecosystem) because it seamless combines HUGO scripts, text blocks (Markdown, R Markdown, LaTeX), and code blocks (R, Javascript, and Python).

When I save my web page, R-Studio generates a deployed web page that is immediately available on a web server local to my laptop (http://localhost:4321). For what it’s worth, I could write all of my pages using just my laptop without the internet. Only changes to my core application require access to the internet.

Markdown and R Markdown

To say the least, you can go nuts with this Markdown stuff. In my view, Markdown does all the easy stuff. Under-the-hood, Markdown simply orchestrates combinations of HTML and CSS – no big deal. Searching the web, you can find all sorts of how-to’s on Markdown. See the Markdown Cheatsheet by Pritchard (Pritchard 2017). R-Studio has released an upgrade to Markdown called R Markdown, which allows scripting.

When using R Markdown, I can add quite a few features. Check out this call out for a Theorem.

Theorem 1 (The Binomial Theorem) The binomial theorem states for real or complex \(a, b,\) and non-negative \(n\),

\[\begin{equation} (a+b)^n = \sum^n_{k=0} {n \choose k} a^{n-k}b^k \end{equation}\]

where \({n \choose k} = \frac {n!}{k!(n-k)!}\) is a binomial coefficient (AoPS 2021).
